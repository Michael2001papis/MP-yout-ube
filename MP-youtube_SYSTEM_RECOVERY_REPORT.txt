--------------------------------------------------------------------------------
1. ROOT PROBLEMS FOUND
--------------------------------------------------------------------------------

A. Global typography (FIXED IN THIS PASS)
   • Legacy Vite-style rules set h1 to very large pixel sizes (56px) and h2 to
     24px with margins. All screens already use Tailwind `text-*` on headings,
     but the global rules increased specificity surface area and could cause
     confusing cascade behavior across browsers or future edits.

B. Global layout (ADDRESSED IN EARLIER COMMITS; VERIFIED STABLE)
   • #app was previously constrained/centered like a demo template; current code
     uses full width, left alignment, flex column, min-height 100svh — correct
     for a video shell.

C. Unused theme tokens in :root / .dark / .warm (FIXED IN THIS PASS)
   • --social-bg and --shadow were defined but never referenced in TSX or CSS
     (dead weight after legacy block removal).

D. Auth + missing Firebase (MITIGATED EARLIER + HARDENED NOW)
   • Setting a global `error` when env vars were missing produced a red banner
     on every route — removed from context; Auth page carries setup copy.
   • Auth methods could theoretically be invoked with `auth === null` when
     Firebase was not configured — now fail fast with assertFirebaseConfigured().

E. Document height chain (FIXED IN THIS PASS)
   • html/body height was implicit only; added html { height: 100% } and
     body { min-height: 100% } for predictable full-viewport stacking with #app.

F. Remaining non-code risks (DOCUMENT ONLY)
   • Firestore Security Rules must enforce hidden videos and admin writes — not
     solvable in the SPA alone (see section 6).
   • React StrictMode in dev can double-invoke effects (e.g. view increments);
     production behavior differs.

2. WHAT WAS FIXED (THIS PASS)
--------------------------------------------------------------------------------

  • src/style.css
      - Replaced global h1/h2 font-size/margin template rules with margin-only

        calls never hit Firebase with a null app when misconfigured.

  • Build: `npm run build` (tsc + vite) — PASS after changes.

5. VISUAL ONLY VS REAL FUNCTIONALITY
--------------------------------------------------------------------------------

  Visual-only (does not enforce security by itself)
    • Route guards (RequireAuth / RequireRole) — hide UI routes; do not stop a
      determined client from calling Firestore with custom code if rules allow.
    • “Admin” derived from VITE_ADMIN_EMAILS — visible in bundle; must match
      server-side policy via Rules.
    • Hidden video message on Watch — UI hides player; document read may still
      be possible if Rules allow public read of hidden docs.

  Real functionality (actual behavior when configured)
    • Firebase Auth sign-in/out, session via onAuthStateChanged.
    • Firestore reads/writes as implemented in services (when rules permit).
    • Storage uploads from Upload page.
    • Forms validated with zod + react-hook-form.
    • Lazy-loaded routes load correct chunks after Suspense.
